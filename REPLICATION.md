# Replication Package: Beyond Text-Only Estimation

This replication package accompanies the paper *"Beyond Text-Only Estimation: Does Combining NLP-Based Requirement Analysis with Repository-Mined Structural Features Improve Feature-Level Effort Prediction?"*

## Overview

This package contains the complete source code, configuration, and instructions needed to replicate the study's experiments. Due to the size of raw data and trained models, those artifacts are regenerated by running the pipeline rather than included directly.

## Requirements

- Python 3.10+
- Git
- ~10 GB disk space (for cloned repositories)
- GitHub Personal Access Token (for data collection)
- Internet connection (for downloading pre-trained models from HuggingFace)

## Quick Start

```bash
# 1. Clone and set up
git clone <this-repo>
cd btoe
python -m venv .venv
source .venv/bin/activate   # Linux/Mac
# .venv\Scripts\Activate.ps1  # Windows PowerShell
pip install -r requirements.txt

# 2. Configure
cp .env.example .env
# Edit .env and add your GitHub Personal Access Token

# 3. Reproduce the full study (estimated: 8-10 hours)
python -m src.pipeline --step collect           # Mine issue-PR pairs (~4-6 hours)
python -m src.pipeline --step clean_data        # Clean dataset
python -m src.pipeline --step data_quality      # Verify data quality
python -m src.pipeline --step nlp_features      # Extract CodeBERT embeddings (~30 min)
python -m src.pipeline --step repo_features     # Extract PyDriller features (~2-4 hours)
python -m src.pipeline --step train             # Train Models A, B, C (~30 min)
python -m src.pipeline --step analyze           # Statistical comparison
python -m src.pipeline --step shap_analysis     # SHAP feature importance
python -m src.pipeline --step error_analysis    # Error analysis plots
python -m src.pipeline --step examples          # Prediction examples

# 4. Reproduce ablation studies
python -m src.pipeline --step sensitivity       # Duration threshold sweep (~2 hours)
python -m src.pipeline --step encoder_ablation  # CodeBERT vs UnixCoder vs BGE (~1.5 hours)
python -m src.pipeline --step feature_selection_ablation  # PCA vs PLS vs Top-K (~45 min)
python -m src.pipeline --step train_model_b_no_numfiles   # Model B without num_files (~5 min)

# 5. Run validation tests
pytest tests/test_pipeline_correctness.py -v
```

## Key Design Decisions

### Target Variable
Elapsed wall-clock time from issue creation to PR merge, measured in hours. This differs from the story-point target used in SE3M (Fávero et al., 2022) because story points are unavailable in public GitHub repositories.

### Feature Extraction
- **NLP features**: Issue title + body only (PR text excluded to prevent data leakage). Processed through CodeBERT to produce 768-dim embeddings, reduced to 50 PCA components, plus 6 derived features.
- **Repository features**: 13 structural features extracted via PyDriller (churn, coupling, file age, change impact).

### SA Computation
Standardized Accuracy follows Shepperd & MacDonell (2012) with 1,000 random sampling runs from the training distribution. Glass's delta effect size is also computed. See `trainer.py:_compute_sa()`.

### Duration Filter
Issues filtered to 1 hour – 90 days. Sensitivity analysis tests robustness across 5 thresholds (14, 30, 60, 90 days, uncapped).

### Random Seed
All stochastic operations use `random_seed=42` from the config for reproducibility.

## Notes on Reproduction

- **Data collection** depends on GitHub API availability and rate limits. Results may vary slightly if repository content has changed since the original study.
- **Bayesian optimization** uses a fixed seed but may produce slightly different hyperparameters across platforms due to floating-point differences.
- **Exact numerical reproduction** of all metrics requires the same Python version (3.11), package versions (see requirements.txt), and operating system.
- **Expected runtime**: Full reproduction takes 8-10 hours on a modern laptop (CPU only). The encoder ablation and sensitivity analysis are the longest steps.
